## Getting Started Path

### Recommended Reading Order for New Users

**Day 1: Installation (30 minutes)**
1. Read [README.md](README.md) - Understand what HyperBox is (10 min)
2. Follow [QUICKSTART.md](QUICKSTART.md) - Get it running (5 min)
3. Run first container from [EXAMPLES.md](EXAMPLES.md#example-1-your-first-container) (15 min)

**Day 2: First Project (45 minutes)**
1. Review [API_REFERENCE.md](API_REFERENCE.md#global-options) - Command basics (10 min)
2. Complete [EXAMPLES.md](EXAMPLES.md#example-2-multi-container-project) - Multi-container (30 min)
3. Check [TROUBLESHOOTING_GUIDE.md](TROUBLESHOOTING_GUIDE.md) - Common issues (5 min)

**Day 3: Deep Dive (1 hour)**
1. Watch [VIDEO_TUTORIALS.md](VIDEO_TUTORIALS.md#video-2-your-first-project-10-minutes) - Video tutorial (10 min)
2. Study [EXAMPLES.md](EXAMPLES.md#example-6-development-workflow) - Dev workflow (20 min)
3. Review [ADVANCED_OPERATIONS.md](ADVANCED_OPERATIONS.md) - Advanced patterns (30 min)

**Ongoing: Reference & Troubleshooting**
- Keep [API_REFERENCE.md](API_REFERENCE.md) handy for command syntax
- Use [TROUBLESHOOTING_GUIDE.md](TROUBLESHOOTING_GUIDE.md) when issues arise
- Reference [EXAMPLES.md](EXAMPLES.md) for patterns and approaches
